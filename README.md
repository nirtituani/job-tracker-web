# JobTracker

A full-stack web application for tracking job applications — built from scratch and deployed to production.

**Live demo:** [job-tracker-web.onrender.com](https://job-tracker-web.onrender.com)

![JobTracker Dashboard](https://img.shields.io/badge/Status-Live-brightgreen) ![Python](https://img.shields.io/badge/Python-3.11-blue) ![React](https://img.shields.io/badge/React-18-61DAFB) ![PostgreSQL](https://img.shields.io/badge/PostgreSQL-Supabase-3ECF8E)

---

## Features

- **Dashboard** — live stats (total, active, interviews, rejected) with animated gradient cards
- **Application table** — search, filter by status, color-coded rows (green = active, gray = rejected)
- **Add / edit / delete** — full CRUD with optimistic UI updates (no full-page refresh)
- **Authentication** — email/password signup and Google OAuth (one-click login)
- **Settings** — customize status labels and colors, "Applied Via" options; persisted in localStorage
- **CSV export** — download all your data anytime
- **Data import** — migrate from a SQLite database via a one-command script
- **Analytics view** — breakdown by status with response-rate summary

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | React 18, Vite, Tailwind CSS |
| Backend | Python / Flask |
| Database | PostgreSQL (Supabase) |
| Auth | Flask sessions, Werkzeug password hashing, Google OAuth (Authlib) |
| ORM / Pooling | SQLAlchemy 2 with QueuePool |
| Deployment | Render (backend + static frontend served from Flask) |

---

## Architecture

```
browser
  └── React SPA (Vite build, served as static files from Flask)
        └── REST API  (/api/*)
              └── Flask + SQLAlchemy
                    └── PostgreSQL (Supabase, transaction pooler)
```

The frontend is built with Vite and served directly by Flask — no separate frontend server in production. A single Render web service handles everything.

Authentication uses cookie-based Flask sessions. Google OAuth is handled server-side via Authlib, with the redirect URI dynamically set from `RENDER_EXTERNAL_URL`.

SQLAlchemy's `QueuePool` keeps persistent connections to Supabase, avoiding the per-request TCP+SSL reconnect overhead that would otherwise add 2–5 seconds of latency.

---

## Performance optimisations

- **Single init round trip** — on page load, one `/api/init` call returns user + applications + stats together instead of 3 sequential requests
- **Optimistic UI updates** — after add/edit/delete, the table updates instantly from the returned row; stats refresh quietly in the background
- **Connection pooling** — SQLAlchemy QueuePool (pool_size=3) reuses database connections across requests

## Local development

**Requirements:** Python 3.11+, Node 18+

```bash
# Clone
git clone https://github.com/nirtituani/job-tracker-web.git
cd job-tracker-web

# Backend
cd backend
pip install -r requirements.txt
DATABASE_URL=<your-postgres-url> SECRET_KEY=dev python app.py

# Frontend (separate terminal)
cd frontend
npm install
npm run dev
```

The React dev server proxies `/api` requests to Flask on port 5001.

---

## Deployment

Configured for one-click deploy on Render via `render.yaml`.

**Environment variables required:**

| Variable | Description |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string (Supabase transaction pooler) |
| `SECRET_KEY` | Flask session secret (auto-generated by Render) |
| `GOOGLE_CLIENT_ID` | Google OAuth app client ID |
| `GOOGLE_CLIENT_SECRET` | Google OAuth app client secret |
| `RENDER_EXTERNAL_URL` | Set automatically by Render |

The build script (`backend/build.sh`) compiles the React app and copies the output into `backend/static/`, which Flask serves as the SPA.

---

## Data migration

If you have an existing SQLite database from a previous version, import it in one command:

```bash
python migrate_sqlite.py
```

The script reads from the old schema (column names `company_name`, `job_title`, `job_match`), maps them to the new schema, and bulk-inserts via the `/api/import` endpoint.
